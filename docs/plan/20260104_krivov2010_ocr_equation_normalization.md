# Krivov2010_arXiv OCR数式置換・正規化プラン

目的
-----
- `paper/pdf_extractor/outputs/Krivov2010_arXiv/result.md` に残る OCR 数式断片を、原文準拠の式ブロックへ置換する。
- 逗点 `,` の用法を精査し、乗算を `\cdot` 等に正規化して数式の意味解釈を安定させる。
- 置換の影響範囲を最小化し、本文の内容・語順は維持する。

対象範囲
---------
- 対象ファイル: `paper/pdf_extractor/outputs/Krivov2010_arXiv/result.md`
- 参照ファイル: `paper/pdf_extractor/outputs/Krivov2010_arXiv/checks/cleanup_report.json`
- 数式の置換・正規化（ブロック数式と OCR 断片の整理）

非対象範囲
-----------
- 本文の要約・言い換え・追加
- 図表の再構成・表データの復元
- OCR 本文の大規模リライト（段落構成の変更等）

前提・入力
-----------
- 原文の式番号・式ブロックは別途提示される（例: OCR参照/原論文の式一覧）。
- `cleanup_report.json` の `remaining_suspect_lines` が空であることを前提に、数式置換に集中する。

作業ステップ
-------------
1. `cleanup_report.json` を確認し、残存のノイズ種別と範囲を把握する。
2. 置換対象の数式（式番号付き/未番号）を抽出し、対応表を作成する。
3. `result.md` で OCR 数式断片を特定し、本文位置に式ブロックを挿入する。
4. 逗点 `,` を文脈判定して正規化する（乗算は `\cdot`、引数区切りは `,` を維持）。
5. 置換後の式番号・参照（Eq. (n)）の整合性をチェックする。
6. 余分な OCR 断片（式の断片行や残骸）を最小限削除する。
7. 変更箇所を再確認し、追加の崩れがないかをレビューする。

判断基準（逗点の正規化）
----------------------
- 乗算と解釈できる箇所は `\cdot` に置換。
- 関数引数/列挙/区切りは `,` のまま保持。
- 迷う場合は変更せず、保留として記録する。

成果物
------
- `paper/pdf_extractor/outputs/Krivov2010_arXiv/result.md` の数式ブロック整備
- 置換対象と残課題が分かる簡易メモ（必要なら `docs/plan/` に追記）

受け入れ基準
------------
- 式番号付き数式が本文位置で式ブロックに置換されている。
- 乗算・区切りの誤解を招く `,` が整理されている。
- 本文の意味改変を伴う修正がない。

リスクと対策
------------
- 式の文脈解釈違い: 置換前後で原文と照合し、保留フラグを残す。
- OCR断片の取り残し: 置換後に `rg` で断片パターンを再検索する。
